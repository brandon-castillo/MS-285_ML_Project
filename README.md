### Motivation:
Can we emulate an ocean state estimate model just using MITgcm outputs using UNet? This is pertinent to my thesis; if we can emulate MITgcm using machine learning, we can run high resolution features over a long period in a much more computationally efficient and feasible way. This will allow experimentation of estimating ocean states in any high resolution/computationally expensive scenario. In this project, I aim to test whether a UNet architecture can simulate the MITgcm barotropic gyre as a simplified test case. This would provide a foundation for emulating more complex ocean models in the future.

### Data:
I will be using MITgcm snapshot outputs. To start, I will be just using 2D fields, composed from zonal velocity `U`, meridional velocity `V`, and sea-surface height `Eta` as input. This data will be in the form of 2D arrays at sequential timesteps. The dataset is dependent upon snapshot intervals and total model run, but with the grid size of the model being fairly small, it will be easy to process. Each snapshot is 62 x 62, each point signifying magnitude and direction of the velocity. As the data is too large to upload directly to this repo, I have uploaded the data as numpy stacks to google drive, and the notebook includes code to pull the data from the follow zip file in the link.

### Model:
I will be using a UNet architecture for emulating the ocean states using MITgcm data as input. This is appropriate because CNNs, and UNet in particular, has been shown in various papers to be effective in emulating the ocean state using MITgcm data. I will be using mean squared error as the loss function to evaluate the model against the MITgcm snapshots at particular time steps. I will also be using a residual model setup, where the model predicts a delta instead of the entire next state, as this is useful for learning changes in the ocean state where adjacent snapshots are quite similar. The model will be trained on the first 80% of the data, and then validated on the remaining 20%. 

### Analysis: 
I will look at MSE of certain timesteps of the model predictions against the MITgcm snapshots. The results will tell me how close the model is able to emulate the ocean state compared to MITgcm over time. As a baseline I can also compare if my model is doing better than just the intial condtions (peristence model) assuming no change. I will also look at the evolution and MSE of individual variables to see how they perform against the MITgcm snapshots. I evaluate the model in two ways. First, comparing single-step validation MSE against a persistence baseline, to see if the model is doing better than just assuming no change. Second, I do an autoregressive run, where I feed the model's own predictions back in as input for the next step, to see how well accuracy is preserved over time, and if there is a blowup in error. I will also visually compare the model outputs against the MITgcm snapshots to see how well the model captures the evolution of the ocean state.
